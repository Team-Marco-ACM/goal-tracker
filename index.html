Quick Commands v

Marco is a dataset versioning CLI for text, CSV, and TSV workflows with immutable snapshots and reproducible pipelines.

  • marco init Initialize .marco metadata in your project.
  • marco upload <file> -c config.json Ingest data and create hashed version.
  • marco list Show timeline with version tags, times, and hashes.
  • marco restore <version> -o training_data.tsv Restore a version for active training.
  • marco export <version> and marco import <archive.tar.gz> Share versions with teammates.
  • marco diff <v1> <v2> Compare row-level or config changes.
  • marco token-analytics <v1> <v2> Measure token drift with KL divergence.
  • marco verify <version> and marco verify-all Prove reproducibility.
  • marco delete <version> or marco rm <version> Remove old versions safely.

1. Goal: Environment Setup - marco init
What it does: Initializes the .marco data versioning directory in your project folder. How it achieves the goal: It sets up the hidden infrastructure required to track datasets safely without moving, combining, or modifying your actual project files.
2. Goal: Immutability and Versioning - marco upload
What it does: Securely ingests text, CSV, or TSV datasets, applies preprocessing config (or guides creation), and generates a cryptographically secure SHA-256 hash. How it achieves the goal: It creates an untamperable snapshot or commit of your data layer.
3. Goal: Timeline Visibility - marco list
What it does: Displays a timeline of all created versions with tags, timestamps, and hashes. How it achieves the goal: Gives a full history view so you do not lose track of which dataset variation you used.
4. Goal: Data Retrieval for Training - marco restore
What it does: Pulls a selected dataset version from Marco storage and writes it into an active workspace file such as training_data.tsv. How it achieves the goal: Bridges secure storage and model training so versions can be swapped in and out easily.
5. Goal: Team Collaboration and Portability - marco export and marco import
What it does: Packages a complete dataset version with hashes and configs into a tar.gz archive, or imports one into another repository. How it achieves the goal: Eliminates machine-specific handoff issues and preserves data integrity in transit.
6. Goal: Granular Transparency - marco diff
What it does: Prints a line-by-line unified diff in terminal between two datasets or preprocessing configs. How it achieves the goal: Shows exactly what rows were added, deleted, or modified for precise preprocessing audits.
7. Goal: Statistical Insight - marco token-analytics
What it does: Uses KL divergence to measure shifts in token probability distributions between versions. How it achieves the goal: Moves beyond simple counts and flags when critical rare terms disappear or dominate between versions.
8. Goal: Provable Reproducibility - marco verify and marco verify-all
What it does: Re-runs the same config on raw data and checks whether the regenerated hash matches the stored hash. How it achieves the goal: Provides mathematical proof of deterministic pipelines and pinpoints the exact row or step if reproducibility fails.
9. Goal: Resource Management - marco delete or marco rm
What it does: Permanently removes a dataset version's files. How it achieves the goal: Recovers storage space while automatically repairing Git-style lineage so the data tree stays consistent after deletion.
